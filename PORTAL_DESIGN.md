# Eklavya Portal Design

> **Design Philosophy**: So simple a high schooler can use it. Hide all complexity. Feel like talking to a smart friend.

---

## Core Design Principles

### 1. Chat-First Interface
- No forms, no dropdowns, no technical jargon
- Just describe what you want in plain English
- Eklavya asks clarifying questions like a friend would

### 2. One Action Per Screen
- Never overwhelm with choices
- Clear primary action button
- Progress feels natural, not technical

### 3. Visual Over Text
- Progress shown with animations, not logs
- Status shown with icons and colors, not words
- Celebrate wins with confetti and sounds

### 4. Templates Over Blank Canvas
- Pre-built starting points for common projects
- "Remix" existing projects
- Never start from zero

### 5. Mobile-Native
- Designed for phones first
- Works great on desktop too
- Notifications that matter

## Design System Summary

### Colors
- **Primary**: Vibrant purple (#7C3AED) - Creative, modern
- **Success**: Green (#10B981) - Done, live, good
- **In Progress**: Blue (#3B82F6) - Building, working
- **Warning**: Amber (#F59E0B) - Attention needed
- **Background**: Near-white (#FAFAFA) - Clean, easy on eyes
- **Dark Mode**: Deep gray (#1F2937) - For night owls

### Typography
- **Headlines**: Bold, friendly (Inter or similar)
- **Body**: Clean, readable
- **Code**: Monospace only when showing code

### Iconography
- Use emojis liberally - they're universal
- Simple line icons for actions
- No complex icons that need explanation

### Spacing
- Generous whitespace
- Cards with rounded corners
- Breathing room everywhere

### Animation
- Subtle transitions
- Progress animations that feel alive
- Celebration moments (confetti on completion!)

---

## Key Metrics to Track

1. **Time to First Project**: How fast can someone go from signup to building?
2. **Completion Rate**: How many started projects get finished?
3. **Return Rate**: Do users come back to build more?
4. **Share Rate**: Do users share their creations?
5. **Template Usage**: Which templates are most popular?

---

## Accessibility

- Works with screen readers
- Keyboard navigation
- High contrast mode
- Reduced motion option
- Voice input support (talk to build)

---

*This design prioritizes simplicity, delight, and empowerment. A high schooler should feel like a wizard when using Eklavya.*
